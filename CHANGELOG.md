## 架构里程碑：v0.6 路由 → v0.9 单文件（真正的"比过去好"）

v0.2–v0.6 是多文件路由架构：常驻 SKILL.md + 懒加载的 `think.md` / `code.md` playbook。

取证发现致命问题：30 个 spine 子 agent 里只有 8 个读过任何 reference 文件，11 个完全没发起 Read——**懒加载的行为规则根本不触发**。于是 v0.6→v0.8 对 think/code.md 的每次修改都是空操作，分数纹丝不动。

v0.9 把 router 折叠成单文件，所有承载行为的指令放进常驻入口。**Sonnet 实测 17/30 → 25/30（+47%），ceiling-cap 桶从全场 0/6 破到 3/6。** 这是迭代里唯一一次结构性跃升，也是"单文件 > 聪明的多文件路由"的直接证据。

## v0.10：全方面优化（SkillOpt Reflect→Select→Gate）

用 SkillOpt 把 `SKILL.md` 当权重优化、yao-meta-skill 约束精简，目标：在**不回归任何桶**的前提下关闭锁定版 v0.9.2 仍存在的对手差距。`evals/optimize_v010.js` 跑了 Reflect（4 分析师诊断 + 1 回归映射）→ Select（梯度裁剪到 2 条高杠杆编辑）→ Gate（2 个对抗 critic 红队逐桶查回归）。

| 编辑 | 关闭的差距 | 逐桶回归预测 | 门控 |
|---|---|---|---|
| **质疑后必须落地**：点名具体替代路径（沿已识别的更深一层往下给，不在原框架塞选项；推断标 🔶 不硬编；前提错先挡再给；纯文字交付物不进此步）| 破天花板 vs 裸 Opus（留出 42% vs 75%）| cw0 · **ceiling+** · brake0 · code0 · slop0 | 2 critic 通过（并入置信度门修订）|
| **决策梯是内部检查**：明确小代码只交付码、不外挂边界旁注（必写的输入校验/错误处理照常）| 精简代码 vs terse（Sonnet 14 vs 17）| cw0 · ceiling0 · **brake+** · **code+** · slop0 | 2 critic 通过（并入脱钩必写校验修订）|

两条都可泛化、作用域钉死、`SLOW_UPDATE` 块未动。**无任何桶预测为负**——这是"全方面领先、避免局部最优"的纸面保证。

**留出差分验证**（task w0a4oxdsc，v0.9.2 vs v0.10，n=12/桶）：综合 **50→55（+5）**；两个目标桶双双关闭——**破天花板 7→10（58%→83%）、精简代码 10→12（83%→100%）**；cw / brake / ai-slop 三桶**零回归**（12/12、12/12、9/9 持平）；paired better 票 **v0.10 26 > v0.9.2 19**。门控逐桶预测与实测一致。数据：`reports/data_validate_v010.json`。

**追加（代码稳健性——"精简同时避免 bug"）：** 留出的「精简代码」桶测的是**精简度，不是 bug 率**——奖励短、惩罚啰嗦，看不见静默 bug。所以 v0.10 又补一条底线：**正确性不为短让路**，会真实咬人的边界（溢出/负数/空/除零/类型/越界）简洁写进代码本身。双轴验证（task wy2z9pirr，v0.10-base vs +稳健，6 道易踩坑题）：**稳健命中 25%→58%（翻倍多）**，精简守住（12→11，一处过度展开），better 票 5>2。诚实：58% 非满分——prompt 把 Sonnet 边界意识拉高但拉不满（能力+prompt 共同上限）。数据：`reports/data_validate_robust.json`。**锁定 `git tag v0.10.0`。**

## v0.11：全面审计 + 全场复测，撤掉一个被实测抓出的局部最优（`git tag v0.11.0`）

全面设计审计（`audit_full.js`，6 审计员逐节 + 强反局部最优 synthesizer + 2 critic）结论：**v0.10 的决策段 / 代码段 / 对手覆盖已近最优、不动**（驳回了 4 条"为改而改"提案）。只放行 2 条候选：

| 候选 | 全场复测（6 臂 n=18，task w22uou083）结果 | 处置 |
|---|---|---|
| **去 AI 腔原则化**（禁词改"写你会对同事当面说的话"+补漏网词） | ai-slop **16→17/18**（近满分，仅次 humanizer 18）| ✅ 保留 |
| **通用性护栏**（解释/教学/总结… 走直接交付、不挑战） | 🚨 把 **cw 从 18 砸到 15**——它让 spine 在含错误前提的写作/教学题上"照做"不挑战（idx2 技术博客）| ❌ **撤回** |

**这是被全场 arena 实测抓出的局部最优**：通用性护栏修了"闲聊误伤"，却伤了 cw（反顺从是 spine 的核心）。更深的真相——**判断价值与普适性是根本矛盾，给护栏让它在通用场景收手，就会在含错前提的任务上也收手。一个判断类 skill 没法既保留判断、又在所有场景零误伤。** 留出/2 臂测都抓不到这个（cw 在那些题里没被触发），只有全场 6 臂 n=18 抓到了——这正是"像以前一样全面测"的价值。

**发布 v0.11.0 = v0.10 + 去AI腔原则化（护栏已撤）。** 同轮 arena 实测（含护栏变体）spine 仍**综合 75/90 全场第一**（领先次席 7 分）、best 票 24 #1、**ceiling 9 全场断层第一**（次席 ponytail 4）、brake 18 满分、perRun **25/25/25 完全稳定**；撤掉护栏后 cw 恢复 18，综合约 78。数据：`reports/data_sonnet_v0.11.json`。

**通用性的诚实结论**：spine 是**判断类专精 skill**——决策 / 选型 / 评审 / 代码 / 写作上实测领先全场；纯闲聊 / 社交 / 简单交付场景它会略端着（通用性探针 misfire ~3/14 vs 裸模型 0），这是判断层的固有取舍，撤护栏是因为护栏的代价（伤 cw）大于收益。**该把 spine 用在它擅长的活上，不当全局闲聊人格。** 不强行用护栏追"全场景零误伤"——那是会反噬核心的局部最优。

## 诚实边界（写在前面，免得自己骗自己）

- 每桶单轮 n=6 噪声大。跨轮稳定的真信号只有两个：**综合 #1** 和 **ceiling-cap #1**。弱桶会在 ai-slop / code 之间随噪声轮换——追单桶=局部最优空转，所以 v0.9.3 后停止迭代。
- 不声称"碾压全部"。在对手主场（humanizer 专做去 AI 腔）单项可能被噪声内反超。可诚实声称的是：**综合 #1 + 最难的 ceiling-cap 桶 #1 + 五桶全部 ≥ 裸模型**。
- 能力曲线（task wrrvn2det）：spine 在 Haiku / Sonnet / Opus **三层全部综合第一**（67% → 79% → 80%）。单文件版连 Haiku 都跟得动——早期路由版「小模型不如 ponytail」是架构问题，不是宿命。Haiku / Opus 为单轮 n=6，破天花板单项优势只用 Sonnet n=18 声称。
