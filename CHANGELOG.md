## 架构里程碑：v0.6 路由 → v0.9 单文件（真正的"比过去好"）

v0.2–v0.6 是多文件路由架构：常驻 SKILL.md + 懒加载的 `think.md` / `code.md` playbook。

取证发现致命问题：30 个 spine 子 agent 里只有 8 个读过任何 reference 文件，11 个完全没发起 Read——**懒加载的行为规则根本不触发**。于是 v0.6→v0.8 对 think/code.md 的每次修改都是空操作，分数纹丝不动。

v0.9 把 router 折叠成单文件，所有承载行为的指令放进常驻入口。**Sonnet 实测 17/30 → 25/30（+47%），ceiling-cap 桶从全场 0/6 破到 3/6。** 这是迭代里唯一一次结构性跃升，也是"单文件 > 聪明的多文件路由"的直接证据。

## v0.10：全方面优化（SkillOpt Reflect→Select→Gate）

用 SkillOpt 把 `SKILL.md` 当权重优化、yao-meta-skill 约束精简，目标：在**不回归任何桶**的前提下关闭锁定版 v0.9.2 仍存在的对手差距。`evals/optimize_v010.js` 跑了 Reflect（4 分析师诊断 + 1 回归映射）→ Select（梯度裁剪到 2 条高杠杆编辑）→ Gate（2 个对抗 critic 红队逐桶查回归）。

| 编辑 | 关闭的差距 | 逐桶回归预测 | 门控 |
|---|---|---|---|
| **质疑后必须落地**：点名具体替代路径（沿已识别的更深一层往下给，不在原框架塞选项；推断标 🔶 不硬编；前提错先挡再给；纯文字交付物不进此步）| 破天花板 vs 裸 Opus（留出 42% vs 75%）| cw0 · **ceiling+** · brake0 · code0 · slop0 | 2 critic 通过（并入置信度门修订）|
| **决策梯是内部检查**：明确小代码只交付码、不外挂边界旁注（必写的输入校验/错误处理照常）| 精简代码 vs terse（Sonnet 14 vs 17）| cw0 · ceiling0 · **brake+** · **code+** · slop0 | 2 critic 通过（并入脱钩必写校验修订）|

两条都可泛化、作用域钉死、`SLOW_UPDATE` 块未动。**无任何桶预测为负**——这是"全方面领先、避免局部最优"的纸面保证。验证：v0.9.2 vs v0.10 留出题差分（见下方状态）。

## 诚实边界（写在前面，免得自己骗自己）

- 每桶单轮 n=6 噪声大。跨轮稳定的真信号只有两个：**综合 #1** 和 **ceiling-cap #1**。弱桶会在 ai-slop / code 之间随噪声轮换——追单桶=局部最优空转，所以 v0.9.3 后停止迭代。
- 不声称"碾压全部"。在对手主场（humanizer 专做去 AI 腔）单项可能被噪声内反超。可诚实声称的是：**综合 #1 + 最难的 ceiling-cap 桶 #1 + 五桶全部 ≥ 裸模型**。
- 能力曲线（task wrrvn2det）：spine 在 Haiku / Sonnet / Opus **三层全部综合第一**（67% → 79% → 80%）。单文件版连 Haiku 都跟得动——早期路由版「小模型不如 ponytail」是架构问题，不是宿命。Haiku / Opus 为单轮 n=6，破天花板单项优势只用 Sonnet n=18 声称。
